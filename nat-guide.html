如何改善 NAT 类型

以下方法按效果由强到弱排序。不同网络环境结果不同，请结合自身情况逐步尝试。

开始前先确认

  • 先用本页检测你当前的 NAT 类型与公网 IP。
  • 确认是否存在双重 NAT（光猫 + 路由器同时拨号或都在路由模式）。
  • 判断是否处于运营商级 NAT（CGNAT）：公网 IP 不在你路由器上、且端口映射无效。

快速清单

  • 确保只有一层路由（避免“双重 NAT”）。
  • 开启路由器 UPnP（对游戏与 P2P 通常有改善）。
  • 为主机设置端口转发或 DMZ（谨慎使用）。
  • 将光猫改为桥接模式，让路由器直接拨号。
  • 向运营商申请公网 IPv4 或启用 IPv6。
  • 避免 CGNAT（运营商级 NAT）。

实施步骤（建议顺序）

1. 排查是否双重 NAT

如果你的光猫和路由器都在做 NAT，基本很难获得理想的 NAT 类型。优先考虑将光猫切换到桥接模式。

  • 光猫改桥接后，由路由器拨号（PPPoE）。
  • 若无法桥接，可将路由器设为 AP 模式减少一层 NAT。

2. 开启 UPnP

UPnP 能让设备自动申请端口映射，是家庭网络最省心的改善方式。启用后重新检测。

  • 开启后，重启路由器与游戏/应用，再次测试。
  • 若路由器有“安全 UPnP”开关，建议开启更安全的模式。

3. 手动端口转发

对特定游戏或应用，可在路由器中设置端口转发。确保你的设备固定内网 IP。

  • 在路由器里给设备绑定静态 DHCP 地址。
  • 只转发应用官方文档要求的端口。

4. 使用 DMZ（谨慎）

DMZ 会把所有端口映射到一台设备，能显著改善连通性，但安全风险更高。

  • 仅在必要时临时启用，测试后关闭。
  • 确保主机开启系统防火墙与自动更新。

5. 申请公网 IP / IPv6

若处于 CGNAT 环境，需要联系运营商申请公网 IPv4。若条件允许，开启 IPv6 也可改善连通性。

  • 询问是否提供“公网 IPv4”或“专线/企业宽带”。
  • IPv6 可绕过部分 NAT 限制，但需应用支持。

6. 设备与网络优化

升级路由器固件、关闭不必要的 QoS 或防火墙规则，有时也能改善 NAT 结果。

  • 更新路由器固件以获得更好的 NAT 兼容性。
  • 避免多层级的“加速器/网关”叠加。

不同平台的检查入口

  • PS5 / Xbox / Switch：在网络设置中查看 NAT 类型或联机状态。
  • Windows PC：先在本页检测，再配合应用内网络诊断工具。
  • 路由器：检查 WAN 口是否拿到公网 IP 与 UPnP 状态。

路由器设置要点

  • 启用 UPnP，避免同时开启多个冲突的端口规则。
  • 端口转发前先绑定静态 DHCP。
  • 能桥接就桥接，不能桥接就 AP 模式。
  • 路由器防火墙策略过严会影响入站回包。

与运营商相关的限制

  • CGNAT 下几乎无法稳定改善 NAT 类型。
  • 部分运营商可付费提供公网 IPv4。
  • IPv6 若可用，对部分应用是更好的选择。

安全提示

  • DMZ 或大范围端口开放会提升暴露面，请谨慎。
  • 仅按需开放端口，并保持系统更新。

注意事项

  • 不同地区、不同运营商策略差异较大，结果以实际检测为准。
  • 对称 NAT（NAT4）一般最难改善，需要运营商配合。
